Bước 1: Kiểm tra commit trước khi merge sai:
git log --oneline
📌 Xác định commit ID trước khi merge, trong ví dụ này là g7h8i9.

Bước 2: Reset branch A về commit trước khi merge sai
Trường hợp 1: Nếu chưa push lên GitLab
git reset --hard ID:g7h8i9
📌 Kết quả: Branch A quay về commit g7h8i9, tất cả thay đổi sau đó bị xóa.

Trường hợp 2: Nếu đã push lên GitLab (phải force push)
git reset --hard ID:g7h8i9
git push origin A --force
📌 Lưu ý: --force sẽ thay đổi lịch sử commit trên GitLab

Bước 3: Merge lại từ master
Sau khi quay lại commit đúng, merge lại master vào và resolve conflict lại từ đầu:
git fetch origin
git merge origin/master
..
